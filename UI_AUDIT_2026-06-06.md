# FreeJoyXConfiguratorQt — UI/UX Audit (2026-06-06)

Whole-app review for design/functionality optimisation, inconsistency, and
duplicated UI rendered in slightly different styles. Read-only audit; no code
changed. Findings grouped by theme, then a tiered action plan.

Reference conventions (what "consistent" means here):
- Accent colours live once in `freejoy_style` (`accentGreen/Red/Amber/Blue`) and
  as `%accent-*%` QSS tokens — nothing should hardcode those hex values.
- Alert/message banners: `freejoy_style::makeAlertBanner`, at the **top** of a
  dialog, ordered red→amber→green (`restackBanners`).
- Disclosure toggles: `freejoy_style::configureSectionToggle` (gear + label + ▸/▾).
- Monochrome icons: themed lucide SVGs via `setThemedIcon`/`tintedSvgPixmap`/`iconInk`.
- Group-box titles: left-aligned, text over frame.
- Shifts & Timers tab (just redesigned) is the reference for grouping/spinbox/units.

## THEME A — Hardcoded colours bypassing the accent tokens  (biggest cross-cutting issue)

Status colours, "active" greens, and pin/bar colours are hand-pasted as hex in
many places. Most break dark-theme tracking and all bypass the single source of
truth in `freejoy_style`.

| # | Where | Issue | Sev | Fix |
|---|---|---|---|---|
| A1 | dialogs: `dfuinstalldialog.cpp:309,351,503,508,709,716`; `flashprogressdialog.ui:36`; `busremapconfirmationdialog.ui:51`; `flashconfirmationdialog.ui:146` | status hex `#27ae60 / #c0392b / #d68910 / #888 / #a05a00 / #D08010` | High | use `accentGreen/Red/Amber` + a muted token |
| A2 | LED: `led.ui:65` (`background:green`) + logical: `buttonlogical.cpp:255` (`QColor(0,128,0)`) | **three different greens** all mean "active/on"; canonical is `%green-fill%`/`accentGreen()` used by `buttonState="on"` + `shiftActive` | High | route both through the existing green role/token |
| A3 | `boarddisplay.h:35` | F103 blue `QColor(0x26,0x82,0xe2)` is byte-for-byte `accentBlue(true)` | Med | call `freejoy_style::accentBlue(true)` |
| A4 | pin boards: `pinsbluepill.ui` / `pinsblackpill.ui` / `pinscontrlite.ui` | ~140 inline pin-pad colours + `color:rgb(0,0,0)` ink repeated; black ink illegible-ish on dark | Med | tokenise the **ink** (pad hues are pictorial — keep), confirm ContrLite palette divergence is intentional |
| A5 | `axes.ui:435-440, 489-494` | Out/Raw bar gradients hardcoded dark-red/olive, ignore `%bar-fill%`/accents | Med | move to tokens/accessors |

## THEME F — Wording / titles

- F1 — Upgrade / Flash / Install used interchangeably for related operations
  (`flashconfirmationdialog.ui:14` "Upgrade Firmware" → `flashprogressdialog.ui:14`
  "Flashing firmware"; `dfuinstalldialog.cpp:67` "Install Firmware" → `:332`
  "Installing firmware"). Pick consistent gerund pairs.

---

## Confirmed-intentional / not-bugs
- `color/` picker hardcoded colours (`Qt::white/black/gray`, inline swatch) — colour
  *is* the data, not a theme accent. Leave (optionally `palette(mid)` for disabled).
- Pin-pad hues mirror the physical board silkscreen legend — tokenise the **ink**,
  not the hues. Confirm ContrLite's different palette is deliberate.
- `buttons/` widgets are the strongest in-scope example of the conventions
  (correct `setThemedIcon`, `setRole`, pulse-fill).

## Suggested tiers

**Tier 1 — quick, high-impact (low risk):**
- C1 apply `role="primary"` to Read/Write (or delete dead block)
- C2 generalise Device-card key/val styling (covers all rows)
- A3 boarddisplay → `accentBlue()`
- A1 dialog status hex → accent helpers
- E3 device-status resting state
- C5/F1 unify "Upgrade Firmware" wording + verbs
- C3/C4 drop dead QSS rule + dead `.ui` styled placeholders

**Tier 2 — consistency sweeps:**
- A2 unify the "active green"
- D1 left-align the centered group titles
- D2/D3 spinbox width/units/label-alignment convention
- B6 kill the duplicate AxesCurves icon-tinting helpers (use `setThemedIcon`)
- B4/B5 collapse the duplicate triangle/banner helpers
- E2/E1 DFU detect-status → banner; drop double-confirm
- E4/E5 LED spinbox affordance + Mono-LED header/wording

**Tier 3 — structural refactors (bigger):**
- B1 shared progress dialog (DFU + flash)
- B2/B3 LED timer cards + per-pin PWM row widget
- B7 shared encoder-row widget
- A4/A6 tokenise pin-board ink + pintypehelper tooltips

**Out of scope to defer:** anything that changes the wire format or behaviour;
the `.ts` catalog refresh still pending from earlier.
